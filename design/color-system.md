# Color System

TradeBill Canada should feel clean, trustworthy, lightweight, and practical. The palette should support fast scanning in the field and professional PDF confidence.

## Core Palette

| Token | Hex | Use |
| --- | --- | --- |
| Turquoise | `#18BFC3` | Primary actions, accent line, selected controls |
| Deep Turquoise | `#0E8F94` | Pressed states, stronger accents |
| White | `#FFFFFF` | Main background, PDF background |
| Off White | `#F7FAFA` | Grouped backgrounds, table header fills |
| Dark Text | `#172326` | Primary text |
| Muted Text | `#6B7A7C` | Secondary text, metadata |
| Border Gray | `#DDE7E8` | Dividers, table borders, form borders |
| Success Green | `#19A974` | Paid, accepted, success states |
| Warning Orange | `#F59E0B` | Draft, overdue warning, partial states |
| Error Red | `#D64545` | Rejected, cancelled, destructive actions |

## UI Style

- White background.
- Turquoise primary buttons.
- Large tap targets.
- Minimal form clutter.
- Clean Apple-native controls.
- Rounded cards for repeated document/client rows.
- Avoid heavy shadows.
- Avoid finance-dashboard visual noise.

## Typography

Use Apple system font:

```text
SF Pro / system default
```

Guidance:

- Large, clear screen titles.
- Compact but readable document cards.
- Avoid tiny financial-table text on mobile.
- Preserve dynamic type compatibility where possible.

## Status Colors

| Status | Suggested color |
| --- | --- |
| Draft | Warning Orange |
| Sent | Muted Text |
| Accepted | Success Green |
| Rejected | Error Red |
| Converted | Deep Turquoise |
| Unpaid | Warning Orange |
| Partially Paid | Deep Turquoise |
| Paid | Success Green |
| Overdue | Error Red |
| Cancelled | Muted Text |

## PDF Styling

- Use white background.
- Use one turquoise accent line.
- Keep table borders subtle.
- Ensure black-and-white print remains readable.
- Do not depend on color alone for totals or statuses.
